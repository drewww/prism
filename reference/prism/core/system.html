System¶

class System : Object¶

A ‘System’ is a class representing a level-wide event handler that can be attached to a Level object. It listens to events such as an actor taking an action, moving, or a tick of time. This should be used for mechanics that affect the entire level; such as hunger, fov, or lighting. For event handlers that apply to a single actor, use a Condition instead. If you want a system that receives all messages from all levels, like tracking the player’s favor with a god attach it to the game instead. A system’s methods should never be mutated at run time, the SystemManager does cacheing on the event handlers here to improve performance.

__new(self: Object, ...: any)¶

The default constructor for the class. Subclasses should override this.

global: boolean¶

A system defined global can only be attached to the Game object. It will see all events from all levels.

name: string¶

A system must define a name that is unique to the System.

requirements: string[]¶

A table of requirements that must be met for the System to be attached to a Level.

softRequirements: string[]¶

A table of optional requirements that ensure proper order if both Systems are attached.

owner: Level?¶

The level that holds this system.

beforeActions: table<Action, fun(level: Level, actor: Actor, action: Action)>¶

A table mapping specific actions to event hooks.

afterActions: table<Action, fun(level: Level, actor: Actor, action: Action)>¶

A table mapping specific actions to event hooks.

initialize(self: System, level: Level)¶

This method is called when the Level is initialized. It is called after all of the Systems have been attached.

Parameters:

level (Level) – The Level object this System is attached to.

postInitialize(self: System, level: Level)¶

This method is called after the Level is initialized. It is called after all of the Systems have been initialized.

Parameters:

level (Level) – The Level object this System is attached to.

beforeAction(self: System, level: Level, actor: Actor, action: Action)¶

This method is called after an actor has selected an action, but before it is executed.

Parameters:

  • level (Level) – The Level object this System is attached to.

  • actor (Actor) – The Actor object that has selected an action.

  • action (Action) – The Action object that the Actor has selected to execute.

afterAction(self: System, level: Level, actor: Actor, action: Action)¶

This method is called after an actor has taken an action.

Parameters:

  • level (Level) – The Level object this System is attached to.

  • actor (Actor) – The Actor object that has taken an action.

  • action (Action) – The Action object that the Actor has executed.

beforeMove(self: System, level: Level, actor: Actor, from: Vector2, to: Vector2)¶

This method is called before an actor moves.

Parameters:

  • level (Level) – The Level object this System is attached to.

  • actor (Actor) – The Actor object that is moving.

  • from (Vector2) – The position the Actor is moving from.

  • to (Vector2) – The position the Actor is moving to.

onMove(self: System, level: Level, actor: Actor, from: Vector2, to: Vector2)¶

This method is called after an actor has moved.

Parameters:

  • level (Level) – The Level object this System is attached to.

  • actor (Actor) – The Actor object that has moved.

  • from (Vector2) – The position the Actor moved from.

  • to (Vector2) – The position the Actor moved to.

onActorAdded(self: System, level: Level, actor: Actor)¶

This method is called after an actor has been added to the Level.

Parameters:

  • level (Level) – The Level object this System is attached to.

  • actor (Actor) – The Actor object that has been added.

onActorRemoved(self: System, level: Level, actor: Actor)¶

This method is called after an actor has been removed from the Level.

Parameters:

  • level (Level) – The Level object this System is attached to.

  • actor (Actor) – The Actor object that has been removed.

afterOpacityChanged(self: System, level: Level, x: number, y: number)¶

Called when an actor or tile has its opacity changed.

Parameters:

  • level (Level) – The Level object this System is attached to.

  • x (number) – The x coordinate of the tile.

  • y (number) – The y coordinate of the tile.

onTick(self: System, level: Level)¶

This method is called every 100 units of time, a second, and can be used for mechanics such as hunger and fire spreading.

Parameters:

level (Level) – The Level object this System is attached to.

onTurn(self: System, level: Level, actor: Actor)¶

This method is called when a new turn begins. The actor is the actor that is about to take their turn.

Parameters:

  • level (Level) – The Level object this System is attached to.

  • actor (Actor) – The Actor object that is about to take its turn.

onTurnEnd(self: System, level: Level, actor: Actor)¶

This method is called when a new turn ends.

Parameters:

  • level (Level) – The Level object this System is attached to.

  • actor (Actor) – The Actor object that is about to take its turn.

onYield(self: System, level: Level, event: Message)¶

This method is called whenever the level yields back to the interface. The most common usage for this right now is updating the sight component of any input controlled actors in the Sight system.

Parameters:

  • level (Level) – The Level object this System is attached to.

  • event (Message) – The event data that caused the yield.

onDescend(self: System, level: Level)¶

This method is called when descending to a lower level.

Parameters:

level (Level) – The Level object this System is attached to.
